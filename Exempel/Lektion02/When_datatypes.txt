List -> När man vill lagra data som ska kunna ändras och man bryr sig om ordningen.
Ett utbud i en butik skulle kunna vara en lista eller en närvarolista.

Tuple -> När man vill ha en mängd data som man vet inte kommer ändras och man bryr sig om ordningen.
Koordinater passar bra som Tuple.

Set -> När man vill lätt kunna jämföra uppsättningar med data och när man vill förhindra dubletter.
Snabbt att konvertera till set och tillbaka för att ta bort dubletter.
Supersnabbt att kolla om något finns i set.


Dicts -> När man har stora mängder data som man vill kunna referera till snabbt.


Behöver du kunna lagra dubletter?
	Isf, List eller Tuple
	
	Behöver du kunna ändra på data?
		List
	Behöver du något som är oföränderligt?
		Tuple

Behöver du kunna ändra på det som ska lagras?
	Lägg det som värden i en Dict.
